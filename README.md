## Overview
InspectAI is a premier inspection intelligence platform designed for property management, restoration, and real estate professionals. It leverages cutting-edge **Gemini 2.5 Flash** vision models to automatically analyze walkthrough videos of properties. It maps the property, detects structural anomalies, identifies code violations, and calculates localized repair cost estimates in real-time.

## Key Features
* **Multimodal Video Analysis**: Upload MP4/MOV videos and let AI analyze the property frame-by-frame.
* **Automated Defect Extraction**: Vision AI instantly extracts structural anomalies and logs physical defects without requiring manual data entry.
* **Instant Repair Liabilities**: Automatically generates a baseline repair cost estimate using historical contracting data.
* **Custom Pricing Context**: Provide your own localized pricing guidelines in the dashboard to fine-tune the AI's repair estimates.
* **Exportable Reports**: Generate beautiful, comprehensive PDF reports containing executive summaries, itemized defect lists, and cost ranges.

## Technology Stack
* **Frontend**: React 19, Tailwind CSS, Lucide Icons, Framer Motion (Vite)
* **Backend**: Node.js, Express
* **AI Engine**: Google GenAI SDK (Gemini 2.5 Flash / Pro)
* **Authentication & Database**: Firebase Auth, Cloud Firestore
* **Storage**: Cloudflare R2 (S3-compatible) for raw video storage
* **Deployment**: Render.com (Monolithic Web Service)

## Deployment Guide

InspectAI is designed as a monolithic full-stack application (the Express server handles the `/api` routes and serves the built React frontend).

### 1. Cloudflare R2 Setup (Video Storage)
Because video files are large, they are uploaded directly from the browser to Cloudflare R2 using pre-signed URLs generated by the backend.
1. Create a Cloudflare R2 bucket (e.g., `inspect-ai`).
2. Generate an API token with **Edit** access.
3. Enable **Public R2.dev URL** (or connect a Custom Domain) for the bucket.
4. **CORS Policy**: You must add the following CORS policy to your R2 bucket settings so the browser can upload files:
```json
[
  {
    "AllowedOrigins": [
      "http://localhost:5173", 
      "https://inspectai-e49p.onrender.com"
    ],
    "AllowedMethods": ["PUT", "GET", "POST", "HEAD"],
    "AllowedHeaders": ["*"]
  }
]
```

### 2. Firebase Setup (Auth & Database)
1. Create a Firebase project.
2. Enable **Authentication** (Google Provider).
3. Enable **Firestore Database**. Make sure to secure it so users can only access their own data.

### 3. Environment Variables
You must supply the following environment variables to your deployment provider (and your local `.env` file):
```env
# Gemini
GEMINI_API_KEY=your_gemini_api_key

# Firebase
VITE_FIREBASE_API_KEY=your_api_key
VITE_FIREBASE_AUTH_DOMAIN=your_auth_domain
VITE_FIREBASE_PROJECT_ID=your_project_id
VITE_FIREBASE_STORAGE_BUCKET=your_storage_bucket
VITE_FIREBASE_MESSAGING_SENDER_ID=your_messaging_sender_id
VITE_FIREBASE_APP_ID=your_app_id

# Cloudflare R2
R2_ENDPOINT=https://<account_id>.r2.cloudflarestorage.com
R2_ACCESS_KEY_ID=your_access_key
R2_SECRET_ACCESS_KEY=your_secret_key
R2_BUCKET_NAME=inspect-ai
R2_PUBLIC_URL=https://pub-your_public_url.r2.dev
```

### 4. Deploying to Render
1. Create a new **Web Service** on Render.
2. Connect your GitHub repository.
3. Set the Build Command: `npm install && npm run build`
4. Set the Start Command: `npm start`
5. Paste all your environment variables into the **Environment** tab.
6. Deploy!
